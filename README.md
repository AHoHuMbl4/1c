# «Второй мозг» на данных 1С

AI-бот, который отвечает на вопросы по данным компании из 1С — **строго на чтение**, ничего в 1С не меняя. Данные 1С регулярно выгружаются во внешнюю базу знаний, индексируются, и бот отвечает с цитатами на источники.

**Статус: работает end-to-end.** На стенде (Бухгалтерия 3.0) спрашиваешь бота в Telegram → он отвечает по реальным данным 1С. Раскатывается копипастом на любую конфигурацию 1С (Бухгалтерия / ERP / УТ …) с разовой настройкой под бизнес.

## Как это работает

```
Windows (1С)                        Ubuntu LXC (весь наш контроль)
┌─────────────────────┐   по IP    ┌──────────────────────────────────────────────┐
│ 1С:база             │  (роутер   │ OData-шлюз :6011  ── только GET (read-only)   │
│  └ IIS (служба)     │   .1)      │   │                                            │
│     └ OData ────────┼───────────►│   ├─ ETL (ночью) ──► md-таблицы ──► KB-репо   │
│        (read-only   │            │   │                                (GitLab)    │
│         ai_reader)  │            │   │                                   │        │
└─────────────────────┘            │   └─ config-ui :6012 (галочки «что тянуть»)   │
                                   │                                       ▼        │
                                   │  braine: oikb → OpenWebUI/pgvector (индекс)    │
                                   │          → Telegram-бот + HTTP API (ответы)    │
                                   └──────────────────────────────────────────────┘
```

- **Канал чтения — штатный OData 1С на IIS** (служба Windows): авто-старт, многопоточно, переживает ребут. Не встроенный MCP-тулкит — он для прода не годится (`docs/TOOLKIT_TRANSPORT_ROOTCAUSE.md`).
- **Read-only — два слоя:** пользователь `ai_reader` (нет прав записи) + шлюз (режет всё, кроме GET).
- **Второй мозг — braine** (RAG-бот без галлюцинаций, отвечает с цитатами): холодный контур (ночная выгрузка ETL → индексация) + горячий (бот отвечает по индексу).
- **Zero-touch:** настроил один раз → работает само, переживает перезагрузки.

Полная картина — [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

## Структура репозитория

```

ubuntu/          всё, что на LXC (наш контроль)
  1c-gateway/      OData read-only шлюз (+ dev: MCP whitelist-прокси)

  1c-config-ui/    веб-галочки «что тянуть» + discovery

```

## Правила проекта
- Вся работа — только внутри этой папки. Каждое изменение — коммит + push в `origin/main`.
- 🔴 **1С только читаем** — инвариант. Ни один запрос не меняет данные 1С.
- На серверах (LXC/Windows) состояние меняем только по явной инструкции владельца.
- Перед любым изменением базы 1С — бэкап (`windows/scripts/backup-1c.ps1`).
- Секреты — в `credentials/` (вне git); на серверах — в `/etc/*.env` (chmod 600).
