# «Второй мозг» на данных 1С

AI-ассистент компании по данным из 1С — **строго на чтение**, ничего в 1С не меняя. Отвечает
в Telegram живым языком: **факты** (контрагенты, договоры, реквизиты) и **аналитику/отчёты**
(суммы, «топ», динамика — таблицей и графиком). Данные 1С регулярно выгружаются во внешние
хранилища, а бот собирает ответ из них.

**Статус: работает end-to-end.** Стенд (Бухгалтерия 3.0): пишешь боту в Telegram → он отвечает по
реальным данным 1С; отчёты приходят таблицей + графиком-картинкой. Раскатывается копипастом на
любую конфигурацию 1С с разовой настройкой под бизнес.

> 🚀 **Развернуть на новом сервере / подключить новую 1С-базу → [`docs/RUNBOOK_DEPLOY.md`](docs/RUNBOOK_DEPLOY.md)**
> (пошагово, воспроизводимо; §10 — слой аналитики и подключение новой базы без хардкода).

## Как это работает — два слоя мозга + диалоговая оболочка

```
Windows (1С)              Ubuntu LXC (весь наш контроль)
┌────────────┐  по IP   ┌───────────────────────────────────────────────────────────────┐
│ 1С:база    │ (роутер  │ OData-шлюз :6011 ── только GET (read-only)                      │
│ └ IIS      │  .1)     │   └ ETL (ночью, выбранные сущности) ──┬─► KB-репо (GitLab)      │
│   └ OData ─┼─────────►│                                        │     └► braine: oikb →   │
│  (ai_reader│          │                                        │        OWUI/pgvector    │
│  read-only)│          │                                        │        (RAG, факты)     │
└────────────┘          │                                        └─► SereneDB (витрина:    │
                        │                                             OLAP+вектор, отчёты) │
                        │                                                                  │
                        │  Диалог: человек → Telegram @бот → OpenClaw (тон, DeepSeek):      │
                        │     ├ report_1c → SereneDB (NL→SQL под ro-ролью + резолвер Qwen)  │
                        │     └ ask_1c    → braine RAG (факты/текст)                        │
                        │     → verify-плагин (ГЕЙТ: числа сверяет КОДОМ; анти-слив КОДОМ)  │
                        │     → ответ клиенту (таблица/график)                              │
                        │  1c-bot-monitor: алерт в Telegram, если что-то упало              │
                        └───────────────────────────────────────────────────────────────┘
```

- **Канал чтения — штатный OData 1С на IIS** (служба Windows): авто-старт, многопоточно, переживает
  ребут. Не встроенный MCP-тулкит — он для прода не годится (`docs/TOOLKIT_TRANSPORT_ROOTCAUSE.md`).
- **Read-only 1С — два слоя:** пользователь `ai_reader` (нет прав записи) + шлюз (режет всё, кроме GET).
- **Два мозга:** **braine** (RAG — факты/текст, «найди по смыслу») + **SereneDB** (витрина — точные
  агрегации/отчёты/графики по ВСЕМ строкам; NL→SQL под **read-only ролью**, семантический резолвер
  на Qwen-эмбеддингах). Эмбеддинги — **только Qwen `text-embedding-v4` @ 1536** (DashScope).
- **OpenClaw** — диалоговая оболочка тона над мозгами (нативно, `@test1c_mcp_bot`; забрал Telegram у
  старого tg-bridge). Отвечает живо, но факты берёт только из инструментов.
- **Анти-галлюцинации — КОДОМ, не промтом** (правило владельца): гейт `verify-плагин` детерминированно
  сверяет числа ответа с данными и режет внутреннее (SQL/пути). Промт держит только тон.
- **Zero-touch:** настроил один раз → работает само, переживает перезагрузки; мониторинг алертит при сбое.

Полная картина — [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

## Правила проекта
- 🔴🔴 Вся работа — **только внутри этой папки `1c/`** (в соседние папки воркспейса не смотреть).
  Каждое изменение — коммит + push в `origin/main`.
- 🔴 **1С только читаем** — инвариант. Ни один запрос не меняет данные 1С (2 слоя + read-only роль SereneDB).
- 🔴 **Правила/ограничения — КОДОМ, не промтом** (промт не держит): анти-галлюцинации, анти-слив,
  read-only — детерминированным кодом/гейтом/ролью, персона — только тон.
- Эмбеддинги — **только Qwen** (как braine), ollama не используем.
- На серверах (LXC/Windows) состояние меняем только по явной инструкции владельца.
- Перед любым изменением базы 1С — бэкап (`windows/scripts/backup-1c.ps1`).
- Секреты — в `credentials/` (вне git); на серверах — в `/etc/*.env` (chmod 600).
