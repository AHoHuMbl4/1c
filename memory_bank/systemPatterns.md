# System Patterns

## Архитектура
> Пока пусто — проект только создан. Описывать здесь: состав конфигурации/расширений, ключевые подсистемы, схемы интеграций.

## Соглашения
- Структура репозитория:
  - `memory_bank/` — контекст проекта (этот банк памяти).
  - остальные каталоги появляются по мере работы (например `src/`, `epf/`, `docs/`) — фиксировать их назначение здесь.
- Имена коммитов: `тип(область): краткое описание` — например `feat(отчёты): отчёт по остаткам`, `docs(memory_bank): обновление контекста`.

## Рабочий процесс (git)
1. Изменение → сразу коммит (маленькие атомарные коммиты).
2. Каждый коммит пушится в `origin/main`.
3. При значимых изменениях обновлять `memory_bank/activeContext.md` и `progress.md` тем же коммитом или следующим.

## Правила работы с целевыми машинами (владелец, 2026-07-22)
- **Бэкапы всегда, на Винде — тоже.** Перед любым изменением на целевой машине (файлы, база, конфигурация 1С) — сначала бэкап затрагиваемого. Для 1С-базы — выгрузка `.dt` или копия файлов базы до изменений.
- Целевые машины: LXC `lxc-claude-1c` (проект) и Windows `10.8.0.58` (1С) — см. `techContext.md`.
