# Веб-настройка ETL: галочки «что тянуть из 1С»

Разовая настройка под конкретный бизнес: страница вытягивает из 1С список всех
непустых сущностей (справочники/документы) с числом записей; человек отмечает
бизнес-разделы (продажи, склад, деньги, контрагенты…) и сохраняет. Дальше ETL
тянет только выбранное — автоматически, ночью.

Зачем: «что важно для мозга» — доменное решение (у каждого бизнеса своё; типовая 1С
содержит сотни служебных классификаторов, которые не нужны). Автоматом их не отличить
→ 5-минутный выбор галочками один раз на бизнес. Код универсален, выбор — конфиг.

## Компоненты
- `oc_discover.py` — параллельный discovery: список сущностей + `$count`, эвристика
  «служебное» для предпометки. Общий модуль (его же зовёт UI).
- `oc_config_ui.py` — веб-страница (stdlib http, порт 6012): галочки, поиск,
  «отметить/снять видимые», сохранение, «Пересканировать 1С».
- Выбор → `/etc/1c-etl-selected.txt` (одна сущность на строку). ETL читает его
  (`ETL_SELECTED_FILE`) как список; приоритет: `ETL_INCLUDE` → этот файл → авто.

## Развёртывание (LXC)
```bash
install -D oc_discover.py /opt/1c-config-ui/oc_discover.py
install -D oc_config_ui.py /opt/1c-config-ui/oc_config_ui.py
cp systemd/1c-config-ui.service /etc/systemd/system/
systemctl daemon-reload && systemctl enable --now 1c-config-ui
# первый скан кэша (~минуты, файловая база):
ETL_ODATA_BASE=http://127.0.0.1:6011 python3 /opt/1c-config-ui/oc_discover.py -o /var/lib/1c-config-ui/entities.json
```
Открыть `http://<lxc>:6012` → отметить бизнес-разделы → «Сохранить». Затем
`systemctl start 1c-etl.service` (или дождаться ночного таймера).

## Замечания
- Скан списка небыстрый на файловой базе (сессии 1С); параллелится (`DISCOVER_WORKERS`).
  На клиент-серверной ERP — быстрее.
- Страница — админ-инструмент настройки; в приватной сети LXC. После настройки можно
  остановить (`systemctl stop 1c-config-ui`) — на работу ETL/бота не влияет.
- Эвристика «служебн/данные» — только подсказка (предпометка), решает человек.
