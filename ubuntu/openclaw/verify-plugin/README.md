# braine-verify — анти-галлюцинационный гейт слоя OpenClaw

Плагин OpenClaw (нативный, `definePluginEntry` + хуки `api.on`). Гарантирует
**кодом, а не промтом**, что бот-оболочка не «дорисовывает» факты поверх ответа нашего
braine («второго мозга»). Это требование владельца: «промты не держат».

## Идея
braine уже отдаёт обоснованный ответ (гейты + цитаты). Риск — LLM бота (DeepSeek),
«оживляя тон», добавит/исказит число (ИНН, сумму, дату, цену, код). Плагин детерминированно
сверяет **жёсткие числовые токены** исходящего сообщения с эталоном braine за этот же ход.

## Как работает (три хука)
1. **`after_tool_call`** — когда бот зовёт MCP-инструмент `ask_1c`, захватываем ответ braine
   как эталон хода (ключ — `runId`; несколько вызовов за ход сливаются).
2. **`message_received`** — запоминаем числа из ввода пользователя (если он сам назвал номер
   заказа/ИНН, эхо этого числа — не галлюцинация).
3. **`message_sending`** — из исходящего текста берём числовые токены (≥ `minDigits` цифр,
   с учётом группировки `7 727 406 020`, `1 234,56`). Каждый обязан быть обоснован эталоном
   braine или вводом пользователя. Решение:
   - все обоснованы → **allow** (отдаём «живой» ответ);
   - есть необоснованные, эталон был и с данными → **replace** на дословный ответ braine;
   - эталон был, но braine сказал «нет данных» → **replace** на безопасную строку;
   - эталона не было и есть выдуманное длинное число (ИНН/счёт/телефон, ≥ `highRiskDigits`) →
     **cancel** (не отправляем).

Чистая логика — в `verify-core.js` (без зависимостей, тест `test-verify.mjs`).
`index.js` — только подключение к хукам и состояние хода в памяти процесса.

## Конфиг (конфиг-нейтрально, дефолты в коде)
`plugins.entries.braine-verify.config`:
- `toolName` (`ask_1c`) — имя MCP-инструмента braine.
- `minDigits` (`4`) — минимальная длина числа для сверки. `3` — строже (ловит 3-значные цены),
  но чаще трогает риторические числа (проценты, «100%»).
- `highRiskDigits` (`7`) — длина числа, которое без эталона считаем выдумкой и режем.
- `noDataMarker` (`[НЕТ ДАННЫХ`) — префикс маркера «нет данных» из `mcp_braine`.
- `noDataReply` — безопасная строка клиенту, когда данных нет.
- `refTtlMs` — TTL эталона хода в памяти.

## Границы (честно)
- Сверяются **числа**. Подмена имени/текста без числа не ловится детерминированно
  (имена — нечёткие; для них работает связка «braine = источник» + замена на эталон при любом
  необоснованном числе). Числовые факты (ИНН/суммы/даты/цены) — самый опасный класс — закрыты.
- 3-значные факты при `minDigits=4` пропускаются (компромисс против ложных срабатываний на
  процентах/годах). Владелец может выставить `minDigits=3`.

## Тест и установка
```bash
node test-verify.mjs                 # оффлайн-юниты чистой логики (15 кейсов)
npm pack --pack-destination /tmp     # собрать tgz
openclaw plugins install npm-pack:/tmp/openclaw-braine-verify-1.0.0.tgz --force
openclaw plugins inspect braine-verify --runtime --json   # проверить, что хуки зарегистрированы
# включить: plugins.allow += "braine-verify"; plugins.entries.braine-verify.enabled = true
```
