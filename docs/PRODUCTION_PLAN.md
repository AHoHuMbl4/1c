# План доведения до продакшена — второй мозг на 1С (zero hardcode)

> Директива владельца (2026-07-24): «никакого хардкода — это бессмысленно; исправить всё до уровня
> продакшена, безальтернативно». Цель: система, которую можно навести на ЛЮБУЮ 1С-базу, и она сама
> сконфигурируется и корректно работает — без имён сущностей/колонок/чисел в коде.

## Принцип (инвариант плана)
Всё выводится из ИСТОЧНИКА, не из предположений: структура — из OData-метаданных и интроспекции схемы;
типы — из `duckdb_columns`; смысл — из эмбеддингов/LLM. **В коде НЕТ:** имён сущностей/колонок, списков
слов, magic-чисел, карт «бизнес-термин→колонка». Платформенные контракты 1С (`Ref_Key`, `IsFolder`,
`$orderby`) — это не хардкод, а контракт (как первичный ключ); их использовать можно.

Статусы: `[ ]` не начато · `[~]` частично есть · `[x]` готово.

## Критерий готовности к продакшену
1. Фазы 1–5 (код) зелёные: ноль имён/чисел/языковых-списков в коде; allow-list-валидатор; критик метрики.
2. Фаза 6: `setup` разворачивает систему на чистую 1С-базу без ручных шагов.
3. **Фаза 7 (гейт): зелёный прогон агентов НА РЕАЛЬНЫХ данных** + хардненинг + soak.
Пункт 7.1 (реальные данные + регистры) — предусловие; всё остальное можно готовить параллельно, но
«продакшн» подписывается только после 7.
