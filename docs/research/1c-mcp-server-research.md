
## Топ-5 бесплатных решений (open source)

### 🥇 1C MCP Toolkit (ROCTUP) — РЕКОМЕНДУЕМ для произвольных запросов
| Параметр | Описание |
|----------|----------|
| **GitHub** | https://github.com/ROCTUP/1c-mcp-toolkit |
| **Звёзды** | 191 |
| **Лицензия** | GPL-3.0 (полностью бесплатно) |
| **Ключевое преимущество** | **Встроенный HTTP-сервер внутри обработки 1С — не нужна публикация на веб-сервере!** |

**Доступные инструменты (из коробки):**

| Инструмент | Описание | Для вашей задачи |
|------------|----------|-----------------|
| `execute_query` | **Выполнение запросов на языке запросов 1С** | ⭐ Главное — произвольные SELECT-запросы |
| `execute_code` | **Выполнение произвольного кода 1С** | ⭐ Любая логика на BSL |
| `get_metadata` | Получение структуры метаданных базы | AI видит все справочники, документы, регистры |
| `get_event_log` | Журнал регистрации | Аудит действий |
| `get_object_by_link` | Получение объекта по навигационной ссылке | Детальная информация |
| `get_access_rights` | Права доступа | Контроль безопасности |
| `get_screenshot` | Скриншот окна 1С (Windows) | Визуальная проверка |
| `restart_1c_session` | Перезапуск сессии | Обслуживание |

**Архитектура:**
- **Вариант 1 (Встроенный сервер):** HTTP-сервер запускается прямо внутри обработки 1С через нативную компоненту (DLL). Python **не нужен**.
- **Вариант 2 (Прокси):** Python-сервер с long polling. Поддержка Docker.

**Требования:**
- 1С:Предприятие 8.2.13+ / 8.3.25+
- Не требуется изменение конфигурации
- Не требуется публикация на веб-сервере (Apache/IIS)
- Не использует COM-соединение

**Быстрый старт:**
```bash
# 1. Скачать готовую обработку MCP_Toolkit.epf из Releases
# 2. Открыть в 1С: Файл → Открыть → MCP_Toolkit.epf
# 3. Нажать "Подключиться" — HTTP-сервер запустится автоматически
# 4. Настроить MCP-клиент (Claude Desktop, Cursor и т.д.)
```

### 🥈 1c_mcp (Владимир Харин) — РЕКОМЕНДУЕМ для гибкости
| Параметр | Описание |
|----------|----------|
| **GitHub** | https://github.com/vladimir-kharin/1c_mcp |
| **Звёзды** | 452 (самый популярный) |
| **Лицензия** | MIT (полностью бесплатно) |
| **Ключевое преимущество** | **Создавайте собственные инструменты для любых задач 1С** |

**Как работает:**
1. Устанавливается расширение 1С (`build/MCP_Сервер.cfe`)
2. Публикуется HTTP-сервис `mcp_APIBackend` на веб-сервере (Apache/IIS)
3. Python-прокси (опционально, но рекомендуется) решает проблемы аутентификации и транспорта
4. AI-клиент подключается к MCP-серверу

**Готовые инструменты (метаданные конфигурации):**
- Получение дерева метаданных (справочники, документы, регистры, модули)
- Структура объектов (реквизиты, табличные части)
- Полнотекстовый поиск по метаданным

**Создание собственных инструментов для произвольных запросов:**

```bsl

    Инструмент = Инструменты.Добавить();
    Инструмент.Имя = "get_custom_report";
    Инструмент.Описание = "Получение произвольного отчета из ERP";
    // ... описание параметров
КонецПроцедуры

Функция ВыполнитьИнструмент(ИмяИнструмента, Аргументы) Экспорт
    Запрос = Новый Запрос;
    Запрос.Текст = "ВЫБРАТЬ ...";  // ← ваш произвольный запрос
    Возврат Запрос.Выполнить().Выгрузить();
КонецФункции
```

**Варианты подключения:**
| Вариант | Прямое к 1С | Через Python-прокси |
|---------|-------------|---------------------|
| Схема | AI ←→ 1С напрямую | AI ←→ Прокси ←→ 1С |
| Аутентификация | Небезопасно (без auth) | OAuth2 + Basic Auth |
| Транспорт stdio | Нет | Да |
| Docker | Нет | Да |

**Инструменты (бесплатная версия):**

| Инструмент | Описание |
|------------|----------|
| `get_metadata_tree` | Дерево метаданных (справочники, документы, регистры) |
| `get_object_structure` | Реквизиты, табличные части, измерения |
| `get_form_structure` | Структура форм |
| `get_configuration_info` | Инфо о конфигурации и платформе |
| `search_code` | Полнотекстовый поиск по коду (BM25) |
| `bsl_syntax_help` | Справка по 180 функциям BSL |
| `execute_query` | **Выполнение запросов 1С (только SELECT)** |
| `validate_query` | Проверка синтаксиса запроса |
| `get_event_log` | Журнал регистрации |

**Особенности:**
- **Автоустановка расширения:** `mcp-1c --install "C:\путь\к\базе"`
- Поисковый индекс строится параллельно (~7 сек на 13000+ модулей)
- Работает с локальными моделями (Ollama, LM Studio)
- Неблокирующий старт (MCP доступен сразу, индекс в фоне)

**Конфигурация MCP-клиента:**
```json
{
  "mcpServers": {
    "1c": {
      "command": "/path/to/mcp-1c",
      "args": ["--base", "http://localhost:8080/hs/mcp-1c"]
    }
  }
}
```

### 4. 1c-rest-mcp (HRYNdev) — через стандартный OData API
| Параметр | Описание |
|----------|----------|
| **GitHub** | https://github.com/HRYNdev/1c-mcp |
| **Звёзды** | Небольшой проект |
| **Лицензия** | Open source |
| **Ключевое преимущество** | **Не требует установки расширений — работает через встроенный OData 1С** |

**Как работает:**
- Подключается к стандартному OData REST API 1С (`/odata/standard.odata`)
- Не нужны расширения, обработки, публикация HTTP-сервисов
- Достаточно включить OData в свойствах конфигурации

**Инструменты:**
- `get_warehouses` — список складов
- `search_products` — поиск номенклатуры
- `get_stock` — остатки
- `get_prices` — цены
- `find_counterparty` — поиск контрагентов
- `get_orders` / `get_order_details` — заказы
- `create_order` — создание заказа

**Настройка:**
```env
ONEC_BASE_URL=http://localhost/your_base/odata/standard.odata
ONEC_LOGIN=admin
ONEC_PASSWORD=your_password
```

## Что такое MCP (кратко)

**Model Context Protocol (MCP)** — открытый стандарт от Anthropic (создатели Claude) для подключения AI-ассистентов к внешним системам данных. Работает по принципу:

1. **AI-бот** получает запрос пользователя на естественном языке
2. **MCP-сервер** предоставляет AI список доступных **инструментов** (tools) с описаниями
3. **AI сам выбирает** нужные инструменты и передаёт параметры
4. **MCP-сервер** выполняет запрос к 1С и возвращает результат
5. **AI формулирует** понятный ответ пользователю

MCP — это "USB-C для AI": один протокол для подключения любой модели к любой системе.
