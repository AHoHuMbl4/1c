# Фаза 2: тест MCP Toolkit на стенде

_Начато 2026-07-22. Стенд: Windows 10.8.0.58, платформа 8.3.27.1786 x86, база `C:\1c\bases\buh_test` (Бухгалтерия 3.0.190.11). Обработка: `C:\1c\distr\MCP_Toolkit_x86_v1.8.0.epf`._

## Что уже проверено (Claude, автоматически через COM)
- Платформа + комьюнити-лицензия + база работают end-to-end.
- **Запрос на языке 1С выполняется через COM-соединение** и возвращает данные (получена строка из `Справочник.Валюты`). Значит read-путь на этой связке платформа/база жив — независимо от MCP Toolkit. COM-коннектор зарегистрирован (`regsvr32 comcntr.dll`, из каталога bin).
- ⚠ Свойства COM `Metadata`/`InfoBaseUsers` через PowerShell отдаются только рефлексией и капризничают — для разведки ролей не полагаемся на COM; ai_reader создаём через GUI Бухгалтерии (BSP-профили).

## Почему нужен GUI-шаг владельца
MCP Toolkit — внешняя обработка; её HTTP-сервер (порт 6003) стартует только когда обработка ОТКРЫТА в сессии 1С и нажата «Запустить сервер». Запустить это headless по SSH нельзя (толстому клиенту нужен рабочий стол). Поэтому — один аккуратный GUI-шаг владельца, дальше всё тестирует Claude по HTTP (curl с самой Винды, firewall не трогаем).

## Стадия 2a — функциональный тест (база пока без пользователей = вход админом)

### Шаг владельца (GUI, один раз)
1. Запусти 1С:Предприятие (толстый клиент) на базе `buh_test`. Если попросит создать первого пользователя — можно пропустить/создать админа без пароля.
2. Файл → Открыть → `C:\1c\distr\MCP_Toolkit_x86_v1.8.0.epf`. При предупреждении «защита от опасных действий» — разрешить (обработка доверенная, наша).
3. На форме обработки:
   - **Режим работы:** «Встроенный сервер».
   - **Порт встроенного сервера:** `6003` (по умолчанию).
   - **Токен доступа:** вставить `1kXTdRC_3hmqYhNi46s8nh0hVEAGI6CbqGOO9fP9i-0` (не нажимать «Сгенерировать»).
   - **Секция «MCP инструменты» — ОТКЛЮЧИТЬ тумблеры:** `execute_code`, `restart_1c_session`, `close_1c_session`, `get_screenshot` (и `submit_for_deanonymization`, если есть). Оставить включёнными: `execute_query`, `get_metadata`, `get_event_log`, `get_object_by_link`, `get_link_of_object`, `find_references_to_object`, `get_access_rights`, `get_bsl_syntax_help`.
   - Нажать **«Запустить сервер»**. Статус должен стать «Сервер запущен (порт 6003)».
4. Написать Claude «сервер запущен» — дальше он тестирует сам.

### Тесты Claude по HTTP (curl с Винды, localhost:6003) — стадия 2a
1. `GET /health` без токена → `{"status":"healthy",...}` — сервер поднялся.
2. `POST /mcp` `tools/list` БЕЗ токена → ожидаем 401 (авторизация включена).
3. `POST /mcp` `tools/list` с Bearer → список инструментов; **`execute_code` в нём отсутствовать** (тумблер).
4. `execute_query` с Bearer, безопасный SELECT (напр. топ-5 из справочника) → возвращает данные. ✔ read-путь через MCP работает на 8.3.27.
5. `execute_code` (отключённый) прямым `tools/call` с безвредным телом → **ожидаем отказ**. Если выполнится — критично: тумблер не enforce'ит, форкаем toolkit и вырезаем инструмент. (Тело безвредное, без записи — базе ничего не грозит даже при выполнении.)

⚠ В стадии 2a под админом НЕ делаем реальных попыток записи (у админа есть права; хоть бэкап и есть). Проверки записи — в стадии 2b под read-only.

## Стадия 2b — read-only ужесточение (ворота перед «боевым» использованием)
1. Создать пользователя `ai_reader` в Бухгалтерии (Администрирование → Настройки пользователей и прав → Пользователи) с профилем «только просмотр/чтение», без прав администрирования и изменения. (Точный профиль подберём вместе — BSP-специфика.)
2. Перезапустить обработку MCP Toolkit в сессии, залогиненной как `ai_reader`.
3. Тесты безопасности §8 плана: попытки записи через execute_query (языку нечем — подтвердить), запись любым способом → «Нарушение прав доступа», проверка журнала регистрации на отсутствие событий записи от ai_reader.

## Сетевой доступ
Сейчас тестируем с самой Винды (`curl localhost:6003`) — firewall не трогаем. Для доступа с LXC/бота позже: открыть 6003 только на нужном интерфейсе + firewall/VPN (bind по умолчанию 0.0.0.0 — сузить). Токен — в `credentials/mcp-toolkit.env`.
