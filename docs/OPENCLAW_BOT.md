## Что это (ПРАВИЛЬНАЯ модель — уточнено владельцем 2026-07-23)
OpenClaw — **НЕ замена** нашего бота (braine), а **надстройка-оболочка тона** поверх него.
braine отвечает точно, но «сухо»; OpenClaw делает диалог живым/человеческим.

Схема:
```
человек пишет → OpenClaw принимает → передаёт НАШЕМУ боту (braine API POST /ask :8090)
   → braine отвечает (факты, цитаты, «не выдумывай») → OpenClaw оживляет тон → клиенту
```
braine = мозг/источник истины. OpenClaw = разговорный слой поверх. Данные 1С OpenClaw
сам НЕ трогает — только braine (через наш read-only канал).

🔴 **Критичный инвариант интеграции:** braine гарантирует «не выдумывать» (гейты: каждая
цифра сверена с источником, обязательные цитаты). Оболочка OpenClaw обязана
**переформулировать ТОЛЬКО то, что вернул braine** — не добавлять и не менять факты/цифры.
Иначе теряется весь смысл точного мозга. Это ложится в персону (`AGENTS.md`) и в выбор
механизма вызова.

Источники: `http://gitlab-real.unde.life/money/opwnclaw-bot` (проектная докум. клиент-ботов
владельца, склонирована `/opt/openclaw`) + движок `github.com/openclaw/openclaw`
(склонирован `/opt/openclaw-engine`). Разворачиваем на LXC `192.168.56.42`.

## Стыковка со «вторым мозгом» — НАТИВНЫЙ ПУТЬ: MCP (из документации движка)
Движок склонирован `/opt/openclaw-engine`. По его докам (`docs/cli/mcp.md`) OpenClaw
**нативно работает как MCP-клиент**: команда `openclaw mcp add <name>` сохраняет внешний
MCP-сервер в `mcp.servers`, и его инструменты **автоматически проецируются боту** (цитата:
«embedded OpenClaw exposes configured MCP tools in normal coding and **messaging** tool
profiles» — messaging = наш Telegram-бот). Транспорты: **stdio** (`--command/--arg/--env`)
или **streamable-http** (`--url --transport streamable-http --header ...`). Есть
`toolFilter.include/exclude`, `probe` (проверка), таймауты.

**План интеграции (весь — штатными средствами OpenClaw = правило №1):**
```
человек → OpenClaw-бот (менеджер) → нативно зовёт MCP-инструмент «спросить 1С»
   → наш тонкий MCP-сервер-обёртка (streamable-http) → braine /ask :8090 (факты+цитаты)
   → бот формулирует живой ответ (персона: отвечать ТОЛЬКО фактами инструмента) → клиенту
```
Шаги: (1) поднять тонкий MCP-сервер поверх braine `/ask` (инструмент `ask_1c(question)`);
(2) `openclaw mcp add second-brain --url http://127.0.0.1:<port>/mcp --transport streamable-http`
в инстанции бота; (3) дополнить персону `AGENTS.md` — когда звать инструмент и что отвечать
строго его данными (см. критичный инвариант выше — не досочинять).

**Правило №1 — требует подтверждения владельца:** MCP-сервер-обёртка над braine — это НАШ
код (внешний сервис, подключаемый ШТАТНЫМ `openclaw mcp add`; не патч/плагин ВНУТРИ
OpenClaw). Это самый нативный способ OpenClaw подключить внешний источник. Но раз правило
строгое — перед реализацией подтвердить у владельца, что тонкий MCP-коннектор к нашему
braine приемлем (это не «кастом поверх OpenClaw», а стандартный MCP-connector).

Альтернативы отвергнуты по докам: `web_fetch` — GET-only и **блокирует внутренние хосты**
(braine на localhost) → не подходит; `memory_search` — статичный слепок, слабее RAG braine.

## 🔴 Анти-галлюцинации на слое OpenClaw — КОДОМ, не промтом (требование владельца)
Владелец: на слое OpenClaw не должно быть галлюцинаций, **и промтом это регулировать нельзя**
(«промты не держат» — тот же принцип, что в braine: точность даёт код). Разобрано по доке
движка (не угадано):
- Финальный ответ клиенту **всегда генерирует LLM бота** (DeepSeek). Нативного «passthrough»
  (отдать ответ braine дословно, минуя LLM) в движке НЕТ → LLM может исказить факт при
  «оживлении». Промт-запрет — не гарантия (владелец прав).
- **Нативная кодовая точка контроля есть:** typed plugin hook **`message_sending`**
  (`docs/concepts/agent-loop.md`) — перехватывает ИСХОДЯЩЕЕ сообщение до отправки;
  `{cancel: true}` терминально отменяет/заменяет. Плюс `before_tool_call`.

**Механизм защиты (принцип braine `verify.py`, но на слое OpenClaw):**
```
бот-LLM формулирует ответ → хук message_sending (наш verify-код):
   сверяет «жёсткие токены» ответа (числа/даты/цены/имена) с выводом braine за этот ход;
   несверенное → заменить на дословный ответ braine / cancel → только тогда отправка
```
Гарантия — детерминированной сверкой кода, не промтом.

⚠️ **ПРАВИЛО №1 — развилка, нужно решение владельца:** хук `message_sending` — это
**typed plugin hook = плагин = кастом**. Нативного НЕ-кастомного способа исключить
галлюцинации кодом в движке нет (passthrough отсутствует, промт запрещён). Значит:
- либо **согласие на плагин-verify** (кастом; даёт кодовую гарантию как braine),
- либо иной путь по указанию владельца.
Без его согласия плагин не пишем (правило №1).

  `.env` (600, `DEEPSEEK_API_KEY`). Gateway = systemd **user**-юнит `openclaw-gateway.service`
